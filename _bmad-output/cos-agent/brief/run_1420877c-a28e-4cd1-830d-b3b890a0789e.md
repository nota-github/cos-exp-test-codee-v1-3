# 오늘의 미니 체크인 - Initial Brief

## Problem / Opportunity
하루를 시작하거나 정리할 때 오늘의 기분, 꼭 해야 할 일, 짧은 메모를 빠르게 남기고 싶어도 메모 앱이나 할 일 앱은 기능이 과하거나 화면이 분산되는 경우가 많다. 이 제품은 한 화면에서 오늘의 상태를 가볍게 기록하고 다시 열었을 때 그대로 이어볼 수 있는 초경량 체크인 경험을 제공하는 데 목적이 있다.

## Target User or Use Case
- 오늘의 상태를 빠르게 정리하고 싶은 개인 사용자
- 별도 가입, 동기화, 백엔드 없이 브라우저에서 바로 쓰고 싶은 사용자
- 출근 전, 공부 시작 전, 하루 마감 시점에 짧게 체크인하려는 사용 상황

## Core Value Proposition
오늘의 기분, 할 일 3개, 짧은 메모를 한 화면에서 바로 기록하고 유지할 수 있는 매우 단순한 개인용 체크인 앱을 제공한다. 경험의 핵심은 빠른 입력, 한눈에 보이는 구성, 새로고침 후에도 유지되는 안정감이다. 화면은 깔끔하고 가벼운 대시보드 느낌을 유지하며, 모바일에서도 자연스럽게 한 화면 흐름이 유지되어야 한다.

## Success Criteria Alignment
- 첫 화면에서 오늘 날짜가 보여야 한다.
- 기분은 3가지 선택지 중 하나로 고를 수 있어야 한다.
- 할 일은 정확히 3개 항목으로 입력 가능해야 하며, 각 항목은 체크 상태를 가질 수 있어야 한다.
- 짧은 메모를 작성할 수 있어야 한다.
- 입력한 기분, 할 일, 체크 상태, 메모는 새로고침 후에도 유지되어야 한다.
- 별도 백엔드나 서버 의존 없이 바로 실행 가능해야 한다.
- 모든 핵심 기능은 한 화면 안에서 보여야 하며, 모바일에서도 읽기와 입력이 자연스러워야 한다.
- 밝고 차분한 톤을 유지하되, 버튼과 입력창은 시각적으로 명확히 구분되어야 한다.

## Initial Scope and Non-Goals
### In Scope
- 오늘 날짜 표시
- 기분 3종 선택
- 할 일 3개 입력 및 체크
- 짧은 메모 작성
- 브라우저 로컬 저장소 기반 상태 유지
- 단일 화면 중심의 반응형 UI

### Non-Goals
- 사용자 계정, 로그인, 클라우드 동기화
- 백엔드, 데이터베이스, API 연동
- 할 일 개수의 동적 추가/삭제
- 날짜별 기록 히스토리, 통계, 공유 기능
- 알림, 협업, 복잡한 생산성 기능

## Uncertainty Notes
현재 기준에서는 기분 3가지의 구체적인 라벨이나 표현 방식은 아직 확정되지 않았다. 이 항목은 이후 PRD 단계에서 사용자 경험 톤에 맞춰 구체화하면 된다.

# Implementation Summary

- `src/lib/formatDate.ts`를 추가해 브라우저 `navigator.language`를 기본 locale로 사용하는 공용 날짜 formatter를 만들었다. 이 모듈만 `Intl.DateTimeFormat`을 직접 사용한다.
- `src/features/header/dateDisplay.ts`를 추가해 `new Date()` 기반의 읽기 전용 헤더 날짜 블록을 만들고, 날짜 문자열 생성은 `formatDate()` 한 경로로 제한했다.
- `src/app/createApp.ts`에서 기존 날짜 placeholder panel을 제거하고, 실제 날짜 표시를 상단 헤더 `page-header__topline`에 연결했다. 이로써 날짜가 입력 컨트롤보다 먼저 보인다.
- `src/styles.css`에서 헤더 날짜 박스와 값 타이포그래피를 추가해 날짜 영역의 시각 위계를 강화했다. 기존 밝고 차분한 대시보드 톤은 유지했다.
- `scripts/validate-story-1.2.mjs`와 `package.json`의 `validate:story-1.2`를 추가해 `typecheck -> build -> story-1.2 구조/스타일/formatter 경로 검사`가 종료 코드와 함께 끝나도록 만들었다.
- `README.md`를 현재 구현 범위와 새 validation 명령 기준으로 갱신했다.

# Risks / Decisions Needed

- 아키텍처 문서에 남아 있는 `한국어 고정 포맷 vs browser locale 기반 포맷`의 최종 제품 결정은 아직 defer 상태다. 이번 구현은 `story-1.2` 범위에 맞춰 browser locale 경로 하나로 일관되게 고정했다.
- 이 샌드박스에서는 `npm run dev -- --host 127.0.0.1 --port 4173`가 `listen EPERM`으로 막혀 실제 브라우저 증거를 수집하지 못했다. 추가 시도가 필요하다면 다음 attempt의 요청은 `코드 수정 없이 실제 로컬 브라우저에서 헤더 날짜와 locale 표시만 guided verification으로 확인`이어야 한다.

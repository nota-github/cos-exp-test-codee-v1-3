# Implementation Summary

- `src/features/memo/memoField.ts`를 추가해 메모 입력 surface를 전용 모듈로 분리했다. `textarea`는 `rows="8"` 기반의 기본 multiline 입력으로 두고, visible label과 supporting hint를 함께 배치해 `story-1.5:SCOPE-1`, `story-1.5:SCOPE-2`, `story-1.5:AC-1`을 직접 반영했다.
- `src/app/createApp.ts`에서 기존 인라인 memo markup을 제거하고 `createMemoField('memo-title')`로 교체했다. 메모 카드 자체는 첫 화면 dashboard grid 안에 그대로 유지해 `story-1.5:GOAL-1`과 `story-1.5:AC-3`을 만족하도록 했다.
- `src/styles.css`에서 `panel--memo`, `memo-field`, `memo-field__label`, `memo-field__hint`, `memo-field__input` 전용 스타일을 추가했다. 메모 카드는 todo 카드와 다른 배경/테두리 톤을 사용하고, textarea는 `min-height: 11rem`과 모바일 `min-height: 9.5rem`, 별도 padding을 갖도록 조정해 `story-1.5:SCOPE-3`, `story-1.5:SCOPE-4`, `story-1.5:AC-2`, `story-1.5:AC-4`를 반영했다.
- `scripts/validate-story-1.5.mjs`와 `package.json`의 `validate:story-1.5`를 추가해 `typecheck -> build -> memo structure/style 검사`가 종료 코드와 함께 끝나도록 만들었다.
- 저장/복원, auto-resize, 리치 텍스트는 이번 story 범위 밖이라 건드리지 않았다.

# Risks / Decisions Needed

- 현재 남은 blocker는 제품 코드가 아니라 브라우저 evidence다. 다음 attempt 또는 운영 단계의 구체적 요청은 `코드 수정 없이 실제 로컬 브라우저에서 story-1.5의 multiline 입력/줄바꿈/스크롤과 desktop-mobile 시각 분리를 guided verification으로 확인`이어야 한다.
- 프로젝트 success criteria에 포함된 새로고침 후 유지(localStorage)는 아직 `epic-2` 범위다. 이번 story는 memo input surface까지만 구현했으므로 persistence 미구현은 의도된 defer이며, 현재 story blocker는 아니다.
